What is Cloud Computing?

Cloud computing is the on-demand delivery of IT resources (computing services) across the internet with pay-as-you-go pricing

You only pay for what you use

Cloud services also expand the traditional IT offerings to include things like Internet of Things (IoT), machine learning (ML), and artificial intelligence (AI)

Because cloud computing uses the internet to deliver these services, it doesn’t have to be constrained by physical infrastructure the same way that a traditional datacenter is



Describing the shared responsibility model


The shared responsibility model defines what entity carries responsibilities under cloud computing

The customer and cloud provider have individual and shared responsibilities


CUSTOMER responsibilities:

The information and data stored in the cloud

Devices that are allowed to connect to your cloud (cell phones, computers, and so on)

The accounts and identities of the people, services, and devices within your organization


CLOUD PROVIDER responsibilities:

The physical datacenter

The physical network

The physical hosts


Responsibilities based on SERVICE MODEL:

Operating systems

Network controls

Applications

Identity and infrastructure



Define Cloud Models

There are THREE main types of cloud models:

Private

Public

Hybrid


PRIVATE clouds are used by a single entity, provide greater control for the company / IT dept with the tradeoff being greater costs and fever benefits of a public cloud

can be hosted in a datacenter either on premise, offsite or even by a third party


PUBLIC clouds are built, controlled and maintained by a third-party cloud provider, and allow for anyone to purchase their services


HYBRID clouds are environments that use both public and private clouds that are interconnected

hybrid clouds allow for a private cloud to surge in increased temporary demand by deploying public cloud resources

hybrid clouds also provide an extra layer of security, and also offer the flexibility to choose what service to keep private / public



a fourth type of cloud is a MULTI CLOUD, which involves multiple cloud providers

this is common for migrating from one cloud provider to another, or using different services from multiple cloud providers


Azure ARC is a set of technologies that helps manage your cloud environment, be it in Azure's public cloud, private cloud, hybrid cloud or multi cloud

Azure also has integration with VMWare - Azure VMware Solution lets customer run their VM workloads in Azure with seamless integration and scalability



Describing the consumption-based model

There are 2 types of expenses to consider when comparing IT ingrastructure models:

CapEx (capital expenditure) vs OpEx (operational expenditure)


CapEx expenses are one-time, upfront expenditures (costs) to purchase / secure tangible resources, like a new nuilding, parking lot, building a datacenter, etc

OpEx expenses is spending money on services or products over time, like leasing a vehicle, signing up for cloud services, renting office space, etc

Benefits of high availability and scalability in the cloud  

Back to Top

When building or deploying a cloud application, two of the biggest considerations are UPTIME (or availability) and the ABILITY TO HANDLE DEMAND (or scale)

High availability focuses on ensuring maximum availability, regardless of disruptions or events that may occur

Uptime guarantees depend on the service being used

Guarantees are defined by SLAs (service level agreements)

Azure SLAs are represented as a percentage, related to the service or application's availability aka UPTIME

Azure SLAs define downtime and what potential credits can be issued if uptime is not met

Azure SLA percentages range from 99%, 99.9%, 99.95 and 99.99% depending on the service

99% SLA means the downtime is 1.6 hrs/week or 7.2 hrs/mth

99.99 SLA means the downtime is 10 minutes a week or 43.2 minutes/mth

each Azure service has its own SLA


Scalability has 2 types - Vertical and Horizontal

Scalability itself is the ability to adjust resources to meet demand, so the customer does not overpay

Vertical scaling refers to increasing or decreasing the capability of resources (more / less processing power like RAM, CPUs, etc)

Horizontal scaling refers to adding or subtracting resources - aka scaling in (fewer resources needed) or scaling out (more resources needed)



Reliability and Predicatbility in the Cloud

Back to Top

Reliability is the ability of a system to recover from failures and continue to function - one of the pillars of the Microsoft Azure Well-Architected Framework

The cloud is decentralized, so reasources can be deployed in regions all over the world

This means that if one region goes down for whatever reason, another region can pick up the slack


Predictability is another part of the Well-Architected Framework

predictability can be focused on performance predictability or cost predictability


Performance predictability focuses on predicting the resources needed to deliver a positive experience for the customer

Autoscaling, load balancing, and high availability are just some of the cloud concepts that support performance predictability


Cost predictability focuses on predicting or forecasting the cost of the cloud spend with tools like Total Cost of Ownership (TCO) or Pricing Calculator

resources can be tracked in real time to find patters and trends to help plan better resource deployments

Manageability in / of the cloud

Back to Top

The cloud offers 2 types of manageability: OF the cloud, and IN the cloud


Management OF the cloud speaks to a customer managing their cloud resouces


Management IN the cloud speaks to a customer managing their cloud resouces AND cloud environment
